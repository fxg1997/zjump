## 核心组件

### 1. API Server

**职责**:
- RESTful API 服务
- WebSocket 终端代理
- 业务逻辑处理
- 权限控制

**技术栈**:
- **框架**: Gin
- **ORM**: GORM
- **语言**: Go 1.21+

**核心模块**:
```
internal/
├── api/            # HTTP API 层
│   ├── handler/    # 请求处理器
│   ├── middleware/ # 中间件
│   └── router/     # 路由配置
├── service/        # 业务逻辑层
├── repository/     # 数据访问层
└── model/          # 数据模型
```

### 2. SSH Gateway

**职责**:
- 标准 SSH 协议接入
- 用户认证（密码/公钥）
- 终端会话管理
- 命令审计

**技术栈**:
- **SSH库**: `golang.org/x/crypto/ssh`

**核心模块**:
```
internal/sshserver/
├── server/         # SSH 服务器
├── auth/           # 认证模块
├── terminal/       # 终端处理
└── types/          # 类型定义
```

### 3. Bastion Core

**职责**:
- 堡垒机核心功能
- 协议适配（SSH/RDP）
- 会话录制
- 命令黑名单

**协议支持**:
- **SSH** (完整支持) - Linux/Unix/网络设备
- **RDP** (计划中) - Windows 远程桌面

**核心模块**:
```
internal/bastion/
├── protocol/       # 协议处理器
│   ├── ssh_handler.go
│   ├── rdp_handler.go (WIP)
│   └── interface.go
├── recorder/       # 会话录制
├── blacklist/      # 命令黑名单
├── parser/         # 命令解析
└── websocket/      # WebSocket 处理
```

### 4. 前端 (Web UI)

**职责**:
- 用户界面
- 实时终端
- 会话回放
- 系统管理

**技术栈**:
- **框架**: React 18
- **UI库**: Material-UI
- **终端**: Xterm.js
- **回放**: Asciinema Player

**核心模块**:
```
ui/zjump-web/src/
```

## 关键技术方案

### 1. SSH Host Key 共享（多实例部署）

**问题**: 
传统多实例部署中，每个实例使用独立的SSH密钥，导致客户端切换实例时收到"主机密钥已更改"警告。

**解决方案**:
所有实例从数据库 `ssh_host_keys` 表加载相同的SSH密钥。

**实现**:
```go
// pkg/sshkey/shared_key.go
func GetOrGenerateSharedHostKey(db *gorm.DB, keyType, keyName string) (ssh.Signer, error) {
    // 1. 从数据库查询密钥
    // 2. 如果存在，解析并返回
    // 3. 如果不存在，生成新密钥并保存
}
```

**优势**:
- ✅ 自动同步，无需手动分发
- ✅ 客户端只需信任一次
- ✅ 支持负载均衡和高可用

详见: [SSH Host Key 管理](DEPLOYMENT.md#ssh-host-key-管理)

### 2. WebSocket 会话保持

**问题**:
WebSocket 是有状态连接，轮询负载均衡会导致连接中断。

**解决方案**:
使用 Nginx `ip_hash` 确保同一客户端始终路由到同一后端实例。

**配置**:
```nginx
upstream api_server {
    ip_hash;  # 关键配置
    server 192.168.1.101:8080;
    server 192.168.1.102:8080;
}
```

### 3. 三维权限模型

**设计**:
```
用户组 (UserGroup) + 主机组 (HostGroup) + 系统用户 (SystemUser) = 权限规则 (PermissionRule)
```

**优势**:
- 灵活的权限组合
- 批量授权
- 继承和覆盖
- 时间范围限制

**示例**:
```
开发组 (UserGroup)
  + 测试环境 (HostGroup)
  + dev用户 (SystemUser)
  = 开发人员可以用dev账号访问所有测试服务器
```

### 4. 会话录制

**方案**: Asciinema 格式

**优势**:
- 文本格式，体积小
- 可回放
- 标准格式，工具多

**实现**:
```go
// internal/bastion/recorder/recorder.go
type Event struct {
    Time   float64 `json:"time"`
    Type   string  `json:"type"`
    Data   string  `json:"data"`
}
```

### 5. 命令黑名单

**实现方式**:
- 实时解析命令
- 正则表达式匹配
- 支持全局和用户级规则
- 阻断或仅告警

**示例规则**:
```
rm -rf  → 危险操作，阻断
dd      → 危险操作，阻断
reboot  → 系统重启，告警
```

## 安全设计

### 1. 认证机制

- **平台用户**: 密码 + 公钥双因素
- **SSH Gateway**: 密码或公钥
- **API**: JWT Token

### 2. 数据加密

- **密码**: bcrypt 加密存储
- **敏感数据**: AES-256 加密
- **传输**: HTTPS/TLS

### 3. 权限控制

- **RBAC**: 基于角色的访问控制
- **时间限制**: 临时授权自动过期
- **审批流**: 工单审批

### 4. 审计日志

- **全量记录**: 所有操作留痕
- **会话录制**: 终端会话回放
- **命令审计**: 危险命令拦截
